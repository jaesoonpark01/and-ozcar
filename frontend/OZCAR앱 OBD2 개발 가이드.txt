현대/기아 신차(특히 E-GMP 기반의 아이오닉 5/6, EV6 등)를 대상으로 **고급 데이터 추출(PID)**과 보안 우회, 그리고 Supabase를 통한 자동 알림 시스템을 구축하는 기술 가이드입니다.

1. 현대/기아 EV 핵심 PID 및 계산 수식
현대/기아 전기차는 표준 모드(01)가 아닌 확장 모드(21)를 주로 사용하며, 헤더 주소는 보통 7E4(BMS)를 사용합니다.

항목	PID (Hex)	수식 (A, B, C...는 응답 바이트)	단위
SOC (Display)	21 01	Bytes[34] / 2.0	%
SOC (BMS Real)	21 01	Bytes[33] / 2.0	%
SOH (Health)	21 05	(Bytes[28] * 256 + Bytes[29]) / 10.0	%
배터리 전류	21 01	((Bytes[13] * 256 + Bytes[14]) - 65535) / 10.0	A
급속 충전 여부	21 01	(Bytes[54] & 0x20) > 0 (Bitwise)	Boolean
참고: 응답 데이터의 인덱스는 어댑터가 반환하는 헤더 포함 여부에 따라 달라질 수 있으므로, AT H1(헤더 온) 상태에서 바이트 위치를 정확히 매핑해야 합니다.

2. 보안 게이트웨이(SGW) 우회 및 어댑터 제어
2020년 이후 현대/기아차는 OBD2 포트를 통한 쓰기(Write/Control) 명령을 차단하는 SGW를 탑재하고 있습니다.

우회 방법 (Hardware Bypass)
SGW 우회 커넥터: 대시보드 안쪽이나 시트 하단에 위치한 SGW 모듈을 찾아, 이를 물리적으로 건너뛰게 해주는 Bypass Cable을 설치해야 합니다. 이렇게 하면 CAN Bus에 직접 명령(문 잠금, 공조 제어 등)을 날릴 수 있습니다.

인증된 어댑터: 단순 ELM327보다는 **OBDLink MX+**나 **vLinker MC+**를 사용하세요. 이들은 전송 속도가 빠르고 현대/기아의 확장 프로토콜(MS-CAN 등)을 더 안정적으로 지원합니다.

제어 명령 예시 (UDS Protocol)
SGW가 우회된 상태에서 문을 잠그는 등의 제어는 UDS(ISO 14229) 서비스를 이용합니다.

Tester Present: 3E 00 (세션 유지용 주기적 발송)

Diagnostic Session Control: 10 03 (확장 진단 세션 진입)

IO Control (예시): 2F [ID] [Option] (특정 액추에이터 제어)

import { serve } from "https://deno.land/std@0.131.0/http/server.ts"

serve(async (req) => {
  const { record } = await req.json() // 새로 삽입된 데이터

  // 조건 1: 배터리 잔량이 20% 미만일 때
  if (record.soc < 20) {
    await sendPushNotification(record.user_id, "배터리 경고", `현재 잔량 ${record.soc}%입니다. 충전이 필요합니다.`);
  }

  // 조건 2: 고장 코드(DTC)가 감지되었을 때
  if (record.dtc_codes && record.dtc_codes.length > 0) {

  }

  return new Response("OK", { status: 200 })
})

async function sendPushNotification(userId: string, title: string, body: string) {
  // FCM(Firebase Cloud Messaging) 또는 Expo Push API 호출 로직
  console.log(`[Push] User: ${userId} | ${title}: ${body}`);

TypeScript
'use client';

import { useEffect, useState } from 'react';
import { supabase } from '@/lib/supabase';
import { Battery, Zap, AlertTriangle, Car } from 'lucide-react';
import { LineChart, Line, XAxis, YAxis, CartesianGrid, Tooltip, ResponsiveContainer } from 'recharts';

export default function Dashboard() {
  const [carData, setCarData] = useState<any>(null);
  const [history, setHistory] = useState<any[]>([]);

  useEffect(() => {
    // 1. 초기 데이터 로드
    const fetchInitialData = async () => {
      const { data } = await supabase
        .from('telemetry')
        .select('*')
        .order('created_at', { ascending: false })
        .limit(20);
      
      if (data) {
        setCarData(data[0]);
        setHistory(data.reverse());
      }
    };

    fetchInitialData();

    // 2. Realtime 구독 설정 (OBD-III 원격 감시 핵심)
    const channel = supabase
      .channel('car-realtime')
      .on('postgres_changes', { 
        event: 'INSERT', 
        schema: 'public', 
        table: 'telemetry' 
      }, (payload) => {
        const newData = payload.new;
        setCarData(newData);
        setHistory((prev) => [...prev.slice(1), newData]);
      })
      .subscribe();

    return () => { supabase.removeChannel(channel); };
  }, []);

  if (!carData) return <div className="p-10 text-center text-white">차량 연결 대기 중...</div>;

  return (
    <div className="min-h-screen bg-slate-900 text-slate-100 p-6">

        <div className="flex items-center gap-2 bg-green-900/30 text-green-400 px-3 py-1 rounded-full text-sm font-medium border border-green-800">
          <div className="w-2 h-2 bg-green-500 rounded-full animate-pulse" /> Live Connected
        </div>
      </header>

      {/* 실시간 주요 지표 */}
      <div className="grid grid-cols-1 md:grid-cols-3 gap-6 mb-8">
        <div className="bg-slate-800 p-6 rounded-2xl border border-slate-700 shadow-xl">
          <div className="flex justify-between items-start mb-4">
            <span className="text-slate-400 font-medium">배터리 잔량 (SoC)</span>
            <Battery className={carData.soc < 20 ? "text-red-500" : "text-green-400"} />
          </div>
          <div className="text-4xl font-black">{carData.soc}%</div>

1. 하이브리드 모바일 수집기 (app/collector/page.tsx)
이 코드는 차량에서 데이터를 읽어와 Supabase로 릴레이하는 핵심 로직을 담고 있습니다.

TypeScript
'use client';

import { useState, useRef } from 'react';
import { supabase } from '@/lib/supabase';
import { Bluetooth, Activity, Send, Terminate } from 'lucide-react';

export default function MobileCollector() {
  const [isConnected, setIsConnected] = useState(false);
  const [log, setLog] = useState<string[]>([]);
  const portRef = useRef<any>(null);
  const readerRef = useRef<any>(null);
  const writerRef = useRef<any>(null);

  // 1. ELM327 연결 (Web Bluetooth 또는 Web Serial)
  const connectDevice = async () => {
    try {
      // 브라우저에서 블루투스 시리얼 포트 요청
      const port = await (navigator as any).serial.requestPort();
      await port.open({ baudRate: 38400 }); // ELM327 기본 속도
      
      portRef.current = port;
      writerRef.current = port.writable.getWriter();
      readerRef.current = port.readable.getReader();
      
      setIsConnected(true);
      addLog("장치에 연결되었습니다.");
      
      // ELM327 초기화 명령 전송
      await sendCommand("AT Z\r");  // 리셋
      await sendCommand("AT SP 6\r"); // 프로토콜 6 (CAN 500kbps) 설정
    } catch (err) {
      console.error(err);
      addLog("연결 실패: " + err);
    }
  };

  // 2. 명령어 전송 및 응답 수신
  const sendCommand = async (cmd: string) => {
    const encoder = new TextEncoder();
    await writerRef.current.write(encoder.encode(cmd));
    
    // 응답 대기 및 파싱 (간략화된 예시)
    const { value } = await readerRef.current.read();
    const response = new TextDecoder().decode(value);
    return response;
  };

  // 3. 현대/기아 EV 데이터 파싱 및 Supabase 전송
  const collectAndSend = async () => {
    if (!isConnected) return;

    // 21 01번 PID 요청 (현대/기아 BMS 데이터)
    const rawResponse = await sendCommand("21 01\r");
    
    // Hex 파싱 (실제 구현 시 문자열 정리 및 바이트 분리 필요)
    // 예: "61 01 01 00 ... 48 ..." -> 48(Hex)는 72(Dec)
    // 현대/기아 SoC 공식: Bytes[34] / 2.0
    const bytes = rawResponse.split(' ');
    const soc = parseInt(bytes[34], 16) / 2.0;
    const temp = parseInt(bytes[12], 16) - 40; // 예시 온도 공식

    addLog(`데이터 추출: SoC ${soc}% | Temp ${temp}°C`);

    // Supabase 전송 (Dashboard와 연동)
    const { error } = await supabase.from('telemetry').insert({
      vin: 'KR-IONIQ6-2024', // 고유 VIN
      soc: soc,
      battery_temp_max: temp,
      dtc_codes: [],
      created_at: new Date().toISOString()
    });

    if (error) addLog("전송 실패: " + error.message);
    else addLog("Supabase 동기화 완료");
  };

  const addLog = (msg: string) => setLog(prev => [msg, ...prev].slice(0, 10));

  return (
    <div className="min-h-screen bg-black text-green-400 p-4 font-mono">
      <div className="border-2 border-green-900 rounded-lg p-4 bg-slate-900">
        <h2 className="text-xl mb-4 flex items-center gap-2">
          <Activity /> OBD-III DATA NODE
        </h2>

        {!isConnected ? (
          <button 
            onClick={connectDevice}
            className="w-full bg-green-600 text-black font-bold py-4 rounded-md flex items-center justify-center gap-2"
          >
            <Bluetooth /> ELM327 장치 페어링
          </button>
        ) : (
          <div className="space-y-4">
            <div className="grid grid-cols-2 gap-2">
              <button 
                onClick={collectAndSend}
                className="bg-blue-600 text-white py-3 rounded-md flex items-center justify-center gap-2"
              >
                <Send size={18} /> 수동 업데이트
              </button>
              <button 
                onClick={() => setIsConnected(false)}
                className="bg-red-900 text-white py-3 rounded-md flex items-center justify-center gap-2"
              >
                <Terminate size={18} /> 연결 해제
              </button>
            </div>
            
            <div className="bg-black p-3 rounded border border-green-800 h-64 overflow-y-auto text-xs">
              {log.map((l, i) => <div key={i}>{`> ${l}`}</div>)}
            </div>
          </div>
        )}
      </div>
    </div>
  );
}

2. 정밀 파싱 로직 구현 (lib/obdParser.ts)
바이트 위치(Index)를 기반으로 전기차 핵심 데이터를 추출하는 유틸리티 함수입니다.

TypeScript
export interface CarTelemetry {
  socDisplay: number;      // 계기판 표시 배터리 잔량 (%)
  socBMS: number;          // 실제 배터리 잔량 (%)
  soh: number;             // 배터리 건강 상태 (%)
  current: number;         // 배터리 전류 (A)
  voltage: number;         // 배터리 전압 (V)
  tempMax: number;         // 배터리 최고 온도 (°C)
  isCharging: boolean;     // 충전 여부
}

export function parseHkmcEvData(rawHex: string): CarTelemetry {
  // 1. 공백 제거 및 줄바꿈 정리, '7E4'와 같은 CAN ID 제거
  const hexArray = rawHex
    .replace(/7E4|10|2[1-9]/g, '') // 헤더 및 행 번호 제거 (간이 처리)
    .trim()
    .split(/\s+/);

  // 2. 바이트 배열로 변환
  const bytes = hexArray.map(h => parseInt(h, 16));

  /**
   * 현대/기아 EV 21 01 서비스 데이터 매핑 (기종별 미세 차이 있음)
   */
  
  // SOC 파싱 (Index 33, 34)
  const socBMS = bytes[33] / 2.0;
  const socDisplay = bytes[34] / 2.0;

  // SOH 파싱 (Index 28-29 또는 48-49 등 기종 확인 필요, 여기선 48-49 기준)
  const soh = (bytes[48] * 256 + bytes[49]) / 10.0;

  // 전류(Current) 파싱: 부호 있는 16비트 정수 (2's complement)
  // (A*256 + B - 65535) / 10.0
  const rawCurrent = (bytes[13] * 256 + bytes[14]);
  const current = (rawCurrent - 65535) / 10.0;

  // 온도 파싱 (Index 40)
  // 보통 Offset -40 적용
  const tempMax = bytes[40] - 40;

  // 충전 여부 (특정 비트 플래그)
  const isCharging = (bytes[54] & 0x10) > 0;

  return {
    socDisplay,
    socBMS,
    soh,
    current,
    voltage: (bytes[15] * 2), // 예시 수식
    tempMax,
    isCharging
  };
}

  return query
  select *
  from charging_stations
  where st_dwithin(location, st_setreid(st_point(user_long, user_lat), 4326)::geography, 5000)

  order by location <-> st_setsrid(st_point(user_long, user_lat), 4326)::geography;

2. 푸시 알림 Edge Function 구현 (index.ts)
이 함수는 유입된 데이터를 분석하여 dtc_codes 배열에 내용이 있을 경우, Firebase Cloud Messaging(FCM) 또는 Expo Push API를 통해 알림을 보냅니다.

TypeScript
// supabase/functions/push-dtc-alert/index.ts
import { serve } from "https://deno.land/std@0.131.0/http/server.ts"
import { createClient } from "https://esm.sh/@supabase/supabase-js@2"

serve(async (req) => {
  const { record } = await req.json(); // INSERT된 데이터

  // 1. 고장 코드가 있는지 확인
  if (!record.dtc_codes || record.dtc_codes.length === 0) {
    return new Response("No DTC detected", { status: 200 });
  }

  // 2. Supabase 클라이언트 초기화 (관리자 권한)
  const supabase = createClient(
    Deno.env.get('SUPABASE_URL') ?? '',
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY') ?? ''
  );

  // 3. 고장 코드 상세 정보 가져오기 (마스터 테이블 참조)
  const { data: dtcDetails } = await supabase
    .from('dtc_master')
    .select('*')
    .in('code', record.dtc_codes);

  // 4. 사용자 푸시 토큰 조회 (사용자 프로필 테이블)
  const { data: userProfile } = await supabase
    .from('profiles')
    .select('push_token')
    .eq('id', record.user_id)
    .single();

  if (userProfile?.push_token) {
    // 5. 알림 전송 (예: FCM 사용 시)
    const alertMessage = dtcDetails?.[0] 
      ? `[${dtcDetails[0].level}] ${dtcDetails[0].description}`
      : `새로운 고장 코드(${record.dtc_codes[0]})가 감지되었습니다.`;

    await fetch('https://fcm.googleapis.com/fcm/send', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `key=${Deno.env.get('FCM_SERVER_KEY')}`,
      },
      body: JSON.stringify({
        to: userProfile.push_token,
        notification: {
          title: "⚠️ 차량 긴급 진단 알림",
          body: alertMessage,
          click_action: "https://your-app-url.com/dashboard/diagnostics"
        },
        data: {
          vin: record.vin,
          dtc: record.dtc_codes[0]
        }
      }),
    });
  }

  return new Response("Alert Processed", { status: 200 });
})

import { useEffect } from 'react';
import { supabase } from '@/lib/supabase';

export const usePushNotification = (userId: string) => {
  useEffect(() => {
    const requestPermission = async () => {
      // 브라우저 Notification API 사용
      const permission = await Notification.requestPermission();
      if (permission === 'granted') {
        // 실제 구현 시 Firebase Messaging 등 SDK를 사용하여 토큰 획득
        const token = "dummy-fcm-token-12345";

A. Supabase RPC (Stored Procedure) 활용
매번 수천 개의 행을 불러와 클라이언트에서 계산하는 것은 비효율적입니다. DB 서버에서 직접 통계를 계산하는 함수를 만듭니다.

SQL
-- 월간 전비 및 주행 요약 통계 함수
create or replace function get_monthly_report(p_vin text, p_month int, p_year int)
returns table (
  total_distance float,
  avg_efficiency float,
  max_temp float,
  charge_count int
) as $$
begin
  return query
  select 
    max(odometer) - min(odometer) as total_distance, -- 주행 거리
    avg(current_power) as avg_efficiency, -- 평균 전력 소모 (가공 필요)
    max(battery_temp_max) as max_temp, -- 최고 배터리 온도
    count(*) filter (where is_charging = true) as charge_count -- 충전 횟수
  from telemetry
  where vin = p_vin 
    and extract(month from created_at) = p_month
    and extract(year from created_at) = p_year;
end;
$$ language plpgsql;

1단계: 월간 리포트용 DB 함수 (Supabase RPC)
단순 주행 거리뿐만 아니라 전기차 성능의 핵심인 평균 전비와 **배터리 스트레스(고온 노출 정도)**를 계산하는 고급 함수입니다. Supabase SQL Editor에 아래 코드를 실행하세요.

SQL
-- 월간 차량 리포트 생성 함수
create or replace function get_monthly_car_report(p_vin text, p_year int, p_month int)
returns json as $$
declare
  report_data json;
begin
  with monthly_stats as (
    select 
      -- 1. 주행 거리 (해당 월의 최대값 - 최소값)
      (max(odometer) - min(odometer)) as distance_km,
      -- 2. 평균 전력 소모량 (kW)
      avg(current_power) as avg_power_kw,
      -- 3. 배터리 건강 상태 (평균)
      avg(soh) as avg_soh,
      -- 4. 급속 충전 횟수 (예시: 전류가 50A 이상인 경우)
      count(*) filter (where is_charging = true and current > 50) as rapid_charge_count,
      -- 5. 최고 배터리 온도
      max(battery_temp_max) as max_temp
    from telemetry
    where vin = p_vin 
      and extract(year from created_at) = p_year
      and extract(month from created_at) = p_month
  )
  select row_to_json(monthly_stats.*) into report_data from monthly_stats;
  
  return report_data;
end;
$$ language plpgsql;
2단계: FCM 프로젝트 생성 및 설정
스마트워치 알림의 교량 역할을 할 FCM 설정 과정입니다.

Firebase 콘솔 접속: Firebase Console에서 프로젝트를 생성합니다.

웹 앱 추가: 프로젝트 설정에서 '웹' 아이콘을 클릭하여 앱을 등록하고 firebaseConfig 객체를 복사합니다.

VAPID 키 생성: '클라우드 메시징' 탭에서 '웹 푸시 설정'의 **Web Push certificates(VAPID Key)**를 생성합니다. (나중에 Next.js에서 사용)

서버 키 확인: Edge Function에서 알림을 쏘기 위해 'Project settings' -> 'Service accounts'에서 비공개 키(JSON)를 내려받거나, 이전 방식의 '서버 키'를 확보합니다.

3단계: Next.js 앱에 FCM 연결 (Client Side)
워치 알림의 핵심은 브라우저가 꺼져 있어도 알림을 받을 수 있는 Service Worker입니다.

Bash
npm install firebase
B. FCM 초기화 (lib/firebase.ts)
TypeScript
import { initializeApp } from "firebase/app";
import { getMessaging, getToken, onMessage } from "firebase/messaging";

const firebaseConfig = {
  // Firebase 콘솔에서 복사한 설정값
  apiKey: "...",
  authDomain: "...",
  projectId: "...",
  messagingSenderId: "...",
  appId: "..."
};

const app = initializeApp(firebaseConfig);

export const requestForToken = async () => {
  const messaging = getMessaging(app);
  try {
    const token = await getToken(messaging, { 
      vapidKey: "YOUR_VAPID_KEY_HERE" // Firebase에서 생성한 키
    });
    if (token) {
      console.log('FCM Token:', token);
      return token;
    }
  } catch (err) {
    console.error('An error occurred while retrieving token. ', err);
  }
};

export const onMessageListener = () =>
  new Promise((resolve) => {
    const messaging = getMessaging(app);
    onMessage(messaging, (payload) => {
      resolve(payload);
    });
  });
C. Service Worker 설정 (public/firebase-messaging-sw.js)

4단계: 스마트워치 알림 연동 테스트
토큰 저장: 사용자가 로그인을 하면 requestForToken()을 호출하여 얻은 토큰을 Supabase의 profiles 테이블(또는 user_tokens)에 저장합니다.

알림 발송: 이전에 만든 Supabase Edge Function에서 고장 코드나 충전 완료가 감지되면, 저장된 토큰을 이용해 FCM API를 호출합니다.

워치 확인:

애플 워치: 아이폰 설정에서 해당 앱의 알림이 허용되어 있고, 워치 설정에서 '아이폰 알림 미러링'이 켜져 있으면 즉시 전달됩니다.

갤럭시 워치: 갤럭시 웨어러블 앱에서 해당 앱의 알림 권한을 활성화합니다.
